# BIMInspect вҖ” Project Context

## What this project does

Detects structural damage (cracks, spallation, corrosion, exposed reinforcement bars, efflorescence) in construction photos using a fine-tuned YOLO11m object detection model and writes every detection into an IFC BIM model as an `IfcAnnotation` with `Pset_DamageInspection`.

## Goals

1. **Automate damage surveys** вҖ” replace manual walkthroughs with camera footage + AI.
2. **Geo-reference every defect** вҖ” convert pixel bounding boxes into 3-D building coordinates.
3. **Enrich the BIM model** вҖ” attach damage severity, type, area, and photos as IFC properties.
4. **Generate inspection reports** вҖ” produce PDF/HTML summaries directly from IFC data.

## Dataset

Training data combines three sources:

1. **Manual labels** вҖ” 2,041 images annotated in Label Studio from CODEBRIM tiles (all 5 classes)
2. **CODEBRIM auto-labels** вҖ” 1,052 XMLs tiled at 640Г—640 with 480 stride (train only)
3. **Roboflow datasets** вҖ” 5 public concrete defect datasets (~16,000 images, train only)

**Labeling workflow:**
1. `extract_for_labeling.py` вҖ” extracts 500 high-density tiles per class from the CODEBRIM zip
2. Label Studio (via `start_labelstudio.bat`) вҖ” manual bounding box annotation
3. `build_dataset.py` вҖ” combines all 3 sources, 8-way augmentation on manual, 80/20 split

**v13 dataset (current):**
| Source | Images |
|---|---|
| Manual labels (val anchor) | 2,041 |
| CODEBRIM auto-tiled | ~8,000 tiles |
| Roboflow (5 datasets) | ~16,000 |
| **Val set** | **manual only (reliable mAP)** |

## Development notes

- CODEBRIM zip has a 4 GB offset bug: every image entry's `header_offset` is inflated by 2ВіВІ bytes. Fix: retry at `header_offset вҲ’ 4294967296` when PK magic not found.
- Model weights go in `models/weights/` вҖ” git-ignored (large files).
- IFC templates in `ifc/templates/` are read-only вҖ” always write enriched copies to `ifc/output/`.
- `detector.py` supports detection mode (`best_detection.pt`) and classification fallback (`best.pt`) with Grad-CAM bbox estimation.
- Tiled inference: `TILE_SIZE=640`, `TILE_STRIDE=480`, cross-tile NMS via `torch.ops.torchvision.nms`.
